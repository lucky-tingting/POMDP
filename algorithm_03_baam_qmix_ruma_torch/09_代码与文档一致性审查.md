# BAAM-QMIX-RUMA PyTorch 代码与文档一致性审查

## 当前结论

`algorithm_03_baam_qmix_ruma_torch` 现在不是只提供算法层框架，而是已经包含可运行 PyTorch 实验 runner：

`run_baam_qmix_ruma_experiment.py`

该 runner 已接入总入口 `run_all.py`，并按统一公共参数导出结果。

## 与文档一致的关键点

- 固定候选风险点集合 `N`。
- 动态高风险集合 `H_t`。
- 决策使用动作前信念。
- ML 软观测与 UAV 硬观测分开。
- 实际动作为无人机-风险点匹配矩阵 `Y_t`。
- 每架无人机每个时间步最多访问 1 个风险点。
- 每个风险点每个时间步最多被 1 架无人机访问。
- 使用 PyTorch GRU 局部 Q 网络。
- 使用 QMIX 单调 mixer。
- 使用 target network。
- 使用 TD target 和 MSE loss。
- 使用 action mask 过滤不可行动作。
- 使用 rolling matching 生成可执行匹配动作。
- 使用 episode/序列经验回放，并在训练 batch 中保留时间轴供 GRU 展开。
- 使用虚拟动作表示无人机当前步不执行真实监测。
- Dec-POMDP 近似实现已经拆分到 `step_01_config_and_protocol.py`。
- 训练奖励已经包含飞行成本，因此滚动匹配阶段不再额外重复扣除飞行成本。
- benchmark 的 `global_state()` 已使用完整训练状态向量 `s_tr=(x,b_bar,q,E,eta)`，其中 `eta` 由逐点降雨表示。
- benchmark 的 `local_observations()` 已改为无人机本地状态加 Top-K 候选点槽位，槽位包含点编号、位置、动作前信念、重要性、距离和可行掩码。
- benchmark 的安全矩阵已加入返航电量安全约束。

## 仍需说明的不一致或未完成

1. 当前已经安装 torch，但当前 torch 是 CPU 版：`torch=2.13.0+cpu`，`cuda_available=False`。不能说 CUDA 版 PyTorch 已经替换成功。
2. 当前 `algorithm_03` 使用内部 benchmark 仿真环境；三算法尚未逐时刻共享完全同一份 rainfall/risk scenario 文件。
3. 当前实现距离/能耗代理约束，不是真实电池物理模型。
4. 返航电量安全已经进入 benchmark action mask；避障、防撞、空域规则尚未完整实现。
5. 如果论文要写完全同场景公平比较，需要把 rainfall、高风险状态转移、固定风险点参数抽成共享 scenario 文件，然后三算法读取同一份 scenario。
